UX Improvement

01. Interaction Flow

iOS 환경에서 온보딩 슬라이드 애니메이션이 의도한 방식으로 작동하지 않는 문제를 확인했습니다. 전환 방식과 타이머 흐름을 조정해 사용자가 화면 이동을 자연스럽게 인지할 수 있도록 개선했습니다.

02. Order Data Flow

쿠폰 또는 메뉴에서 선택한 상품 정보가 장바구니 화면까지 유지되도록 데이터 흐름을 구성했습니다. 상품명, 옵션, 가격, 수량 정보를 LocalStorage에 저장하고 페이지 이동 후에도 선택 상태를 확인할 수 있도록 구현했습니다.

03. Mobile Touch UX

닫기, 뒤로가기, 하단 탭, 주문 버튼 등 주요 인터랙션 요소의 터치 영역을 최소 44px 이상으로 확보했습니다. 모바일 환경에서 작은 버튼으로 인한 오터치 가능성을 줄이고 조작성을 개선했습니다.

04. Visual Alignment

헤더 좌우 아이콘 구성 차이로 중앙 타이틀이 시각적으로 밀려 보이고, 장바구니 배지 위치가 어긋나는 문제를 확인했습니다. 좌우 기준 영역과 배지 위치를 조정해 헤더의 시각적 균형을 개선했습니다.

01. 프로젝트 목표

Figma 프로토타입만으로는 상품 선택, 쿠폰 적용, 장바구니 상태 유지, 결제 예정 금액 계산처럼 실제 주문 과정에서 발생하는 상태 변화를 검증하기 어렵다고 판단했습니다.

따라서 HTML/CSS/Vanilla JS를 활용해 모바일 앱처럼 동작하는 웹 프로토타입으로 확장했고, 사용자가 메뉴를 탐색하고 상품을 담는 흐름이 자연스럽게 이어지는지 직접 테스트했습니다.

02. UI/UX 설계

  • 온보딩, 메인, 메뉴, 쿠폰, 장바구니 등 주요 화면 플로우 설계
  • 공통 헤더와 하단 네비게이션 구조를 정의해 앱 전체의 일관성 유지
  • 하단 네비게이션 중심의 모바일 앱 구조 구성
  • 주문 방식 선택을 위한 바텀시트 UI 설계
  • 상품 선택, 쿠폰 사용, 장바구니 확인까지 이어지는 사용자 흐름 구성
  • 빈 장바구니, 선택 상태, 결제 예정 금액 등 상태별 UI 고려

03. 웹 프로토타입 구현 및 트러블슈팅

  • 쿠폰/메뉴에서 선택한 상품명, 옵션, 가격, 이미지 정보가 장바구니 화면에 전달되도록 구성
  • LocalStorage를 활용해 페이지 이동 후에도 장바구니 데이터가 유지되도록 구현
  • 체크된 상품 기준으로 결제 예정 금액이 계산되도록 장바구니 상태 UI 구성
  • 헤더와 하단 탭이 콘텐츠를 가리지 않도록 페이지별 여백과 레이아웃 조정
  • iOS 환경에서 온보딩 슬라이드 애니메이션이 의도대로 작동하지 않는 문제 수정
  • 주요 버튼의 터치 영역을 44px 이상 확보하여 모바일 조작 편의성 개선
  • 헤더 중앙 타이틀과 장바구니 배지 위치가 어긋나는 문제를 수정해 시각적 균형 개선
  • HTML Validator 기준으로 이미지 경로, heading 구조, ARIA 속성, SVG 태그 오류를 점검하고 수정

04. 개선 결과

정적인 앱 시안에서 끝나지 않고, 사용자가 메뉴를 탐색하고 쿠폰을 적용해 장바구니에 담는 흐름까지 확인할 수 있는 모바일 앱형 웹 프로토타입을 완성했습니다.

이를 통해 화면 구성뿐 아니라 데이터 전달, 상태 유지, 모바일 조작성, 브라우저별 동작 차이까지 함께 검증할 수 있었습니다.

05. PRD (요구사항 정의서)

프로젝트 기획 및 주요 요구사항, 화면별 기능 명세를 정리한 PRD 문서입니다.

FitCheck Project Overview

스마트 자세 교정 밴드라는 가상의 제품을 설정하고, 장시간 앉아서 일하는 사용자들이 자신의 자세 문제를 인지하기 어렵다는 점에서 출발했습니다.
